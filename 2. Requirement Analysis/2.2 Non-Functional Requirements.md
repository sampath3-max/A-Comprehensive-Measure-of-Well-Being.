# Non-Functional Requirements

Non-functional requirements describe the quality, performance, and usability expectations of the system.

## NFR-01: Usability

The application should provide a simple and user-friendly interface so that users can easily enter the required socio-economic indicators.

## NFR-02: Performance

The system should generate the HDI prediction within a reasonable response time after the user submits the input.

## NFR-03: Reliability

The system should provide consistent prediction results when the same input values are provided.

## NFR-04: Maintainability

The project should be organized into separate components for:

- Dataset
- Machine Learning model
- Flask application
- User interface

## NFR-05: Portability

The application should be capable of running on systems that support Python and the required dependencies.

## NFR-06: Scalability

The system should be capable of future improvements, such as using larger datasets and additional Machine Learning algorithms.

## NFR-07: Security

The application should process user inputs appropriately and prevent invalid data from being used for prediction.
