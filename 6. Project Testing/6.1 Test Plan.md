# Test Plan

## Objective

The objective of testing is to verify that the HDI Prediction System works correctly and produces the expected results.

## Testing Areas

### 1. Environment Testing

Verify that all required Python packages are installed correctly.

### 2. Model Testing

Verify that the Machine Learning model is trained successfully and generates predictions.

### 3. Input Testing

Verify that the application accepts valid socio-economic indicator values.

### 4. Application Testing

Verify that the Flask application starts and the home page loads correctly.

### 5. Integration Testing

Verify that the Flask application successfully communicates with the trained Machine Learning model.

### 6. Output Testing

Verify that the predicted HDI Score and its corresponding classification are displayed correctly.

## Testing Approach

The system will be tested by:

1. Running the model training script.
2. Starting the Flask application.
3. Entering different input values.
4. Submitting the prediction form.
5. Checking the generated HDI Score.
6. Verifying the displayed HDI category.

## Expected Outcome

All major components of the HDI Prediction System should function correctly and produce the expected results.
