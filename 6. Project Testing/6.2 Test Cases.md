# Test Cases

The following test cases are used to verify the functionality of the HDI Prediction System.

| Test ID | Test Case | Input / Action | Expected Result |
|---|---|---|---|
| TC-001 | Open the application | Open the application URL | Home page loads successfully |
| TC-002 | Display input form | Open the home page | All required input fields are displayed |
| TC-003 | Enter valid values | Enter numerical socio-economic indicators | Values are accepted |
| TC-004 | Submit prediction | Submit the completed form | Prediction is generated |
| TC-005 | Model prediction | Provide valid input values | HDI Score is displayed |
| TC-006 | HDI classification | Generate a prediction | Corresponding HDI category is displayed |
| TC-007 | Result page | Submit valid input | Result page is displayed |
| TC-008 | Model loading | Start the Flask application | Saved model is loaded successfully |
| TC-009 | Invalid input | Enter invalid or non-numerical values | Input should be rejected or handled appropriately |
| TC-010 | Application execution | Run the Flask application | Application runs without errors |

## Test Result Status

The status of each test case can be recorded as:

- **Pass** — Expected result is achieved.
- **Fail** — Expected result is not achieved.

## Expected Outcome

The test cases should confirm that the HDI Prediction System correctly accepts user input, generates HDI predictions, classifies the results, and displays the output.
